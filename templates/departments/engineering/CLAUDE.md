# 開発

## 役割

技術ドキュメント・設計書・デバッグログを管理する。
技術的な意思決定を記録し、再現性と引き継ぎを担保する。

## ファイル構成

```
engineering/
├── CLAUDE.md
├── docs/        ← 技術ドキュメント・設計書
└── debug-log/   ← バグ・問題の調査ログ
```

## ルール

- 技術ドキュメント: `docs/テーマ.md`
- デバッグログ: `debug-log/YYYY-MM-DD-問題名.md`
- 設計書は「概要・設計方針・詳細」の構成にする
- バグ修正時は「再発防止」を必ず記入する
- 技術的な意思決定は `secretary/notes/` に意思決定ログとして残す

## ステータス定義

**デバッグ**: open → investigating → resolved → closed

## 技術ドキュメント形式

```markdown
---
title: ""
created: YYYY-MM-DD
---

# [タイトル]

## 概要

## 設計方針

## 詳細

## 参考・関連リンク
-
```

## デバッグログ形式

```markdown
---
issue: ""
status: open
created: YYYY-MM-DD
---

# [問題のタイトル]

## 症状

## 期待する動作

## 再現手順
1.

## 調査
### 仮説
-
### 発見
-

## 解決策

## 再発防止
```
