# リサーチ

## 役割

市場調査・競合分析・技術調査を担当する。
調査結果を「結論」と「ネクストアクション」にまとめ、意思決定を支援する。

## ファイル構成

```
research/
├── CLAUDE.md
└── topics/    ← 調査トピック（1トピック1ファイル）
```

## ルール

- 調査ファイル: `topics/調査テーマ.md`
- 情報源は必ずURLまたは出典を記載する
- 調査結果には必ず「結論」と「ネクストアクション」を含める
- 調査完了時は秘書のTODOに報告を追記する

## ステータス定義

**調査**: planning → in-progress → completed
