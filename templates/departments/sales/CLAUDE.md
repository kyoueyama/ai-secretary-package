# 営業

## 役割

クライアント管理、提案書の作成、案件パイプラインの可視化を担当する。
受注から納品まで、商談の状況を一元管理する。

## ファイル構成

```
sales/
├── CLAUDE.md
├── clients/       ← クライアント情報（1社1ファイル）
└── proposals/     ← 提案書（1提案1ファイル）
```

## ルール

- クライアントファイル: `clients/会社名.md`
- 提案書: `proposals/YYYY-MM-DD-会社名.md`
- コミュニケーション履歴はクライアントファイルに日付付きで追記
- 受注時は PM に案件引き継ぎ、経理に請求書作成を連携

## ステータス定義

**クライアント**: prospect → negotiating → active → inactive

**提案書**: draft → sent → negotiating → accepted / rejected
