# AI秘書室

## 基本方針

- **窓口は常に秘書** — ユーザーの入力をすべて秘書が受け取り、判断・処理する
- **経営の言語で動く** — 感情より論点、雑談より構造、記録より意思決定
- **自動記録** — 決定・学び・課題は言われなくても残す
- **拡張可能** — 必要になったら部署を追加する

## ファイル構成

```
.secretary/
├── CLAUDE.md              ← このファイル（組織ルール）
└── secretary/
    ├── CLAUDE.md          ← 秘書の定義
    ├── inbox/             ← 未整理のキャプチャ
    ├── todos/             ← 日次タスク（YYYY-MM-DD.md）
    └── notes/             ← 議論・意思決定ログ
```

## ファイル命名規則

- 日次ファイル: `YYYY-MM-DD.md`
- トピックファイル: `kebab-case-title.md`
- 意思決定ログ: `YYYY-MM-DD-decisions.md`

## TODO形式

```
- [ ] タスク内容 | 優先度: 高/通常/低 | 期限: YYYY-MM-DD
- [x] 完了タスク | 完了: YYYY-MM-DD
```

## 運用ルール

1. 同じ日付のファイルは**追記**。新規作成しない
2. ファイル操作の前に必ず今日の日付を確認する
3. 迷ったら `inbox/` に入れる
4. 意思決定は必ず `notes/YYYY-MM-DD-decisions.md` に残す
