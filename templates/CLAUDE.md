# AI秘書室

## 基本方針

- **窓口は常に秘書** — すべての入力を秘書が受け取り、判断・処理する
- **判断を速くする** — 情報を構造化し、選択肢と根拠をセットで提示する
- **記録を残す** — 決定・会議・気づきは自動で保存する
- **拡張可能** — 事業フェーズに合わせて部署を追加する

## ファイル構成

```
.secretary/
├── CLAUDE.md              ← このファイル（組織ルール）
└── secretary/
    ├── CLAUDE.md          ← 秘書の定義・振る舞い
    ├── inbox/             ← クイックキャプチャ（未整理）
    ├── todos/             ← 日次タスク（YYYY-MM-DD.md）
    └── notes/             ← 意思決定ログ・会議録・思考整理
```

## ファイル命名規則

| 種類 | 形式 | 例 |
|------|------|-----|
| 日次タスク | `YYYY-MM-DD.md` | `2026-03-23.md` |
| 意思決定ログ | `YYYY-MM-DD-decisions.md` | `2026-03-23-decisions.md` |
| 会議録 | `meeting-YYYY-MM-DD-相手名.md` | `meeting-2026-03-23-田中.md` |
| 思考整理 | `kebab-case-title.md` | `pricing-strategy.md` |
| 気づき | `YYYY-MM-DD-learnings.md` | `2026-03-23-learnings.md` |

## TODO形式

```
- [ ] タスク内容 | 優先度: 高/通常/低 | 期限: YYYY-MM-DD
- [x] 完了タスク | 完了: YYYY-MM-DD
```

## 運用ルール

1. 同じ日付のファイルは**追記**。新規作成しない
2. ファイル操作の前に必ず今日の日付を確認する
3. 迷ったら `inbox/` に入れる
4. 意思決定は必ず `notes/YYYY-MM-DD-decisions.md` に残す
5. 会議後は決定事項・宿題・次回アクションを必ず記録する
